## Background

Could we give clinical analysts a tool to model how protocol changes ripple through cost projections across a patient population? That was the bet. Getting it right required a frontend that could handle uncertainty — model outputs were _distributions_, not point estimates.

![The Population Cost Forecast view — scenario filters, KPI summary cards, expenditure trend chart, and a variance-flagged provider table](/images/work/cost-of-care-ai.png)

## My role

I joined when it was a Figma prototype. I owned frontend architecture end-to-end:

- React + Redux Toolkit project structure and component contracts
- Okta OIDC integration via a backend token proxy
- Recharts visualisations for distribution rendering, including a custom confidence-interval chart
- CI pipeline with Playwright and Vitest

## Modelling probability in UI state

A scenario had a lifecycle: draft → submitted → processing → complete → stale. I modelled this as a finite state machine in Redux with typed discriminated unions, so the UI could exhaustively switch on `scenario.status` and TypeScript caught missing cases at compile time.

```typescript
type ScenarioState =
  | { status: "draft"; config: ScenarioConfig }
  | {
      status: "processing";
      config: ScenarioConfig;
      jobId: string;
      progress: number;
    }
  | { status: "complete"; config: ScenarioConfig; results: ScenarioResults }
  | {
      status: "stale";
      config: ScenarioConfig;
      results: ScenarioResults;
      reason: string;
    };
```

## Outcome

Shipped to 200+ internal analysts in Q2 2024. Median time-to-insight dropped from ~45 minutes to 7.5 minutes. We passed a formal WCAG 2.1 AA audit at launch — the first product in the division to do so.
