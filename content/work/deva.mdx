## Background

This tool replaced a manual care-management workflow: clinical teams submitting data via email, reviewers working through spreadsheet checklists. It brought that process into a structured interface with validation, audit trails, and role-appropriate views.

## The permission model

Six distinct roles, each with different read/write permissions and navigation. Rather than hardcoding role checks, I built a `usePermissions` hook that returned a typed object from decoded Okta JWT claims:

```typescript
interface Permissions {
  canSubmit: boolean;
  canReview: boolean;
  canAudit: boolean;
  canManageUsers: boolean;
  visibleQueues: QueueType[];
}
```

Components checked `permissions.canReview`, never `user.role === "supervisor"`. When a new role was added post-launch, one mapping change propagated everywhere.

## Form architecture

40+ field types with conditional logic — show field B only if field A exceeds a threshold; require field C only for certain categories. I built a JSON-driven schema on React Hook Form so the product team could change form behaviour without touching component code.

```typescript
type FieldSchema = {
  key: string;
  type: "text" | "select" | "date" | "number" | "file";
  validation?: ValidationRule[];
  displayWhen?: ConditionalRule;
};
```

## Outcome

Replaced a three-year-old email process. Care managers saved 30–45 minutes per review cycle. Compliance reviews that previously required digging through email chains now resolved against a timestamped audit log.
